2D位图用像素点组成图形 特点:不可以放大或缩小，颜色细腻
注意

1.画布宽度和高度只能通过js 或者标签属性来赋值
2.样式要写在绘制之前才能改变绘制的样式
3.绘制的东西不会超过画布的大小

代码 说明 -
dom.getContext("2d") 获取画布对象 -
ctx.strokeRect(x,y,w,h) 绘制空心矩形 x,y矩形左上角位置 w,h 短形宽度和高度 样式如下
ctx.strokeText(str,x,y) 绘制空心文本 x,y文本左上角位置 str 文案 样式如下
—-ctx.strokeStyle = "#47227b" 设置空心样式 空心样式
—-ctx.lineWidth = 1 设置空心宽度 空心宽度
ctx.fillRect(x,y,w,h) 绘制实心矩形 x,y矩形左上角位置 w,h 短形宽度和高度 样式如下
ctx.fillText(str,x,y) 绘制实心文本 x,y文本左上角位置 str 文案 样式如下
—-ctx.fillStyle = "#47227b" 设置实心填充样式 实心样式
—-ctx.globalAlpha = 0.5 透明度 通用
—-ctx.font 字体与大小 字体
—-ctx.textBaseline = "alphabetic" 指定文本基线位置[“alphabetic”,”top”,”bottom”,Middle，Hanging]
基线/上/下/中/悬挂 正常思维使用bottom。默认是alphabetic		 -
—-ctx.measureText(str) 获取此时绘制 str 文本的宽度对象 {width:} -
ctx.clearRect(x,y,w,h) 清除一个矩形范围内所有元素 xy位置,wh范围 -
ctx.moveTo(x,y) 移动到指定点-只移动不画 -
ctx.lineTo(x,y) 画到指定点-又移又画 -

ctx.closePath(x,y) 闭合当前路径.从终点画到起点 -

ctx.translate(x,y) 将画布轴点(原点)移到指定位置,默认是画布中心 -
ctx.save(x,y) 保存画笔状态 -
ctx.restore(x,y) 恢复画笔状态 -

1-算法复杂度的含义

2022-10-24

前言

原文 https://blog.csdn.net/A_dg_Jffery/article/details/99713579

正文

首先o(1), o(n), o(logn), o(nlogn)是
用来表示对应算法的时间复杂度,
这是算法的时间复杂度的表示。不仅仅用于表示时间复杂度，
也用于表示空间复杂度。

时间复杂度是指执行这个算法所需要的计算工作量
空间复杂度是指执行这个算法所需要的内存空间

时间和空间都是计算机资源的重要体现，而算法的复杂性就是体现在运行该算法时的计算机所需的资源多少；
O后面的括号中有一个函数，指明某个算法的耗时/耗空间与数据增长量之间的关系。其中的n代表输入数据的量。

时间复杂度的优劣对比
常见的数量级大小：越小表示算法的执行时间频度越短，则越优；
O(1) < O(logn) < O(n) < O(nlogn) < O(n2) < O(n3) < O(n^2) < O(n!) < O(n^n) // n的n方

1 说明 算法代表
O(1) 常数阶,最低的时空复杂度，也就是耗时与输入数据大小无关 哈希算法
O(logn) 对数阶,当数据增大n倍时，耗时增大logn倍, 2 <= logn < n 二分查找
O(n) 线性阶,数据量增大几倍，耗时也增大几倍。比如常见的遍历算法 遍历算法
O(nlogn) 线性对数阶,n乘以logn，当数据增大256倍时，耗时增大256*8=2048倍 归并排序
O(n^2) 平方阶,数据量增大n倍时，耗时增大n的平方倍 冒泡排序

O(1)

哈希算法就是典型的O(1)时间复杂度，无论数据规模多大，都可以在一次计算后找到目标。

O(logn)

当数据增大n倍时，耗时增大logn倍（这里的log是以2为底的，比如，当数据增大256倍时，耗时只增大8倍，是比线性还要低的时间复杂度）。二分查找就是O(logn)的算法，每找一次排除一半的可能，256个数据中查找只要找8次就可以找到目标。 (个人理解：logn = n除以几次二)

O(n)

O(nlogn)

就是n乘以logn，当数据增大256倍时，耗时增大256*8=2048倍。
归并排序就是O(nlogn)的时间复杂度。

O(n^2)

数据量增大n倍时，耗时增大n的平方倍，这是比线性更高的时间复杂度。冒泡排序，就是典型的O(n x n)的算法，对n个数排序，需要扫描n x n次。
