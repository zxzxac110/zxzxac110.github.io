前言

nvm是一个nodejs的版本管理工具。通过它可以安装和切换不同版本的nodejs

原文 https://www.cnblogs.com/gaozejie/p/10689742.html

下载完毕 傻瓜操作 一路点击下去 记得点击 i accept the agreement
不要用中文路径

指令

说明 示例
nvm list [available] 查看本地安装的所有版本；有可选参数available，显示所有可下载的版本。 list可简化为ls。 npm list
nvm install 版本 安装，命令中的版本号可自定义，具体参考命令1查询出来的列表 nvm install 11.13.0
nvm use 版本 使用特定版本,切换到特定版本 nvm use 11.13.0
nvm uninstall 版本 卸载特定版本 nvm uninstall 11.13.0
nvm version 显示nvm版本。version可简化为v。 npm -v
nvm arch 显示node是运行在32位还是64位。
nvm install [arch] 安装node， version是特定版本也可以是最新稳定版本latest。可选参数arch指定安装32位还是64位版本，默认是系统位数。可以添加–insecure绕过远程服务器的SSL。
nvm on 开启node.js版本管理。
nvm off 关闭node.js版本管理。
nvm proxy [url] 设置下载代理。不加可选参数url，显示当前代理。将url设置为none则移除代理。
nvm node_mirror [url] 设置node镜像。默认是https://nodejs.org/dist/。如果不写url，则使用默认url。设置后可至安装目录settings.txt文件查看，也可直接在该文件操作。
nvm npm_mirror [url] 设置npm镜像。https://github.com/npm/cli/archive/。如果不写url，则使用默认url。设置后可至安装目录settings.txt文件查看，也可直接在该文件操作。
nvm use [version] [arch] 使用制定版本node。可指定32/64位。
nvm root [path] 设置存储不同版本node的目录。如果未设置，默认使用当前目录。

使用nvm 升级 npm 版本

1.去官网找到要升级到的npm 对应的node版本下载
https://nodejs.org/zh-cn/download/releases/
比如升级npm到 v7.21.0 下载 node-v16.6.0-win-x64.zip

2.将下载的安装包里的 node_modules/npm 替换到 当前node版本对应的 node_modules/npm

3.npm -v 查看

属性说明

名称 说明 用法
animation-name 选中你要使用的动画名称 a
animation-duration 设置动画的执行时间 10s
animation-timing 时间曲线函数 同下
linear 动画从头到尾的速度是相同的
ease 默认。动画以低速开始，然后加快，在结束前变慢
ease-in 动画以低速开始
ease-out 动画以低速结束
ease-in-out 动画以低速开始和结束
cubic-bezier(n,n,n,n) 在 cubic-bezier 函数中自己的值。可能的值是从 0 到 1 的数值
animation-delay 动画的延迟时间 0s
animation-iteration-count 指定动画的播放次数 具体数字/infinite
animation-direction 指定关键帧的播放顺序 同下
normal 默认值，0%~100% 正放
reverse 100%~0% 倒放
alternate 轮流播放 先正后反 奇数次正着播放，偶数次反着播放
以上简写方式
animation animation-name animation-duration animation-timing animation-delay animation-iteration-count animation-direction e 5s linear 0.1s infinite normal
最简写方式 animation:e 5s;
animation-fill-mode 设置动画在播放前和播放后的显示状态 动画在延时的时间内，和播放完成之后的时间，是不显示的 同下
backwards 动画播放之前的延迟时间内，动画保持在第一帧上
forwards 动画播放完成，将保持在最后一帧的状态
both 是backwards和forwards的结合体
none 默认值，不填充
animation-play-state 指定动画是处于播放还是暂停的状态上 paused/running

sass

一.简介

1.安装

node-js 8.1 以上安装 32位sass

  • 在线安装scss 在 cmd 控制台打印命令

cnpm install node-sass –save-dev //安装node-sass
cnpm install sass-loader –save-dev //安装依赖包sass-loader
cnpm install style-loader –save-dev //安装style-loader
npm install sass-loader@7.3.1 –save-dev

2.文件转化

单文件转换 node-sass scss/01.scss css/01.css 当前路径下的scss中的01.scss 转换为当前目录下的css中的01.css
多文件转换 node-sass scss(文件夹名称) -o css(文件夹名称) 当前路径下的scss文件夹中的.sass文件 转换为当前目录下的css文件夹中的.css文件
单文件监听 node-sass -w scss/01.scss css/01.css 01.scss内容发生改变，保存的时候，自动转换为01.css
多文件夹监听 node-sass -w scss -o css

3.变量声明

1-变量声明在{}外，整个scss文件都可以使用这个变量
2-变量声明在{}内，只用当前样式可以使用此变量
3-变量重复声明，后声明的变量会覆盖之前声明的，尽量保证变量名称不重复
4-!default规则：如果此变量在之前已经声明赋值了,那么就使用之前的值，如果此变量之前没生命赋值，就使用现在的值

4.声明函数 / 声明混合器 @mixin @include

声明函数 / 声明混合器 @mixin
声明函数 / 声明混合器 @function 可以有return 值
调用函数 / 调用混合器 @include

5.导入

如果一个scss文件以下划线开头，那么这个文件就是一个局部scss文件,在转换成css文件的时候，不会把局部文件进行转换
一般情况，我们会把局部文件引入到全局文件(不以下划线开头的)中。
导入的语法 @import"dropdown";
文件名叫做 _dropdown.scss
导入的时候，不写_,不写后缀
局部文件被导入后，局部文件中样式，会在全局文件转换的css中生成。同时局部文件声明的变量，可以在全局文件中使用。

一个选择器，可以继承另一个选择器的所有样式
