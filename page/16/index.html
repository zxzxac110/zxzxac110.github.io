场景：
1.点击 - 用户未安装 -> 安装 -> 登录 -> 默认跳转到首页 -> 获取参数a=1
2.点击 - 用户已安装 -> 跳转 -> 用户未登录 -> 登录 -> 默认跳转到首页 -> 获取参数a=1
3.点击 - 用户已安装 -> 跳转 -> 用户已登录 -> 用户在首页 -> 获取参数a=1
4.点击 - 用户已安装 -> 跳转 -> 用户已登录 -> 用户不在首页 -> 跳转首页 -> 获取参数a=1

实现：
1.h5端进行跳转 app://index?a=1 并且复制该链接
2.确保首页是永远存在的。在首页 使用 getLinksStream().liste 监听链接。解决3，4场景
3.在首页initState中使用 await getInitialLink() 获取获取应用程序启动时接收到的链接，进行判断获取参数a=1。解决2场景
4.在首页使用 getClipboardData 获取剪切板内容，进行判断获取参数a=1。解决1，2场景（确保与实现3的逻辑不冲突）
