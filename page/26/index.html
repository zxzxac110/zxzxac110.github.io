7-ORM工具

2022-10-24

1.简介

Django中最推荐的数据库操作方式不是直接向MySQL发送SQL语句，而是使用ORM框架技术： Object Relation Mapping, “对象-关系型数据库-映射”技术：

表 面对对象的类
表名 类名 class User
字段, 每个字段有类型 类中有属性: name:string 代表name属性是字符串类型
记录: 可以有很多条具体的数据 实例化的对象: 每个类都可以实例化出很多对象
对象关系映射(ORM): 把 面向对象的类 映射成 表的结构, 然后操作表

ORM框架的三种实现方式：

(1)手工编写映射文件：需要手工创建类(Object)，再手工创建所有的表(Relation)，最后手工指定哪个类的哪个属性 对应 哪个表的哪个列
(2)正向生成Relation方式：手工编写所有的类(Object)，然后使用ORM工具根据类自动生成所有的关系表(Relation)
(3)逆向生成Object方式：手工编写所有的表(Relation)，然后使用ORM工具逆向创建所有的类(Object)
提示：Django框架支持方式2和方式3，

2-服务和依赖注入

2022-10-24

服务和依赖注入

Component 组件，仅应该只负责视图，只参与展示；与展示无关的语句都应该剥离出去；例如：做日志、计时、网络数据访问…
组件=视图数据+事件监听函数
Service 服务，就是一个简单的对象，负责执行从组件中剥离的与视图无关的操作，例如：做日志、计时、网络访问….
服务=日志+计时+网络请求+格式验证

在构造方法中声明需要依赖某个对象，且该对象是“可以被注入的(@Injectable)”,那么Angular就会自动创建依赖的对象，并注入给当前构造方法
Angular中的依赖注入(DI)是根据构造方法的形参类型来识别的！
不放在普通实例方法中；也不根据变量名来识别！

创建服务： ng generate service 服务名
ng generate service fuWu
新增2个文件

CREATE src/app/fu-wu.service.ts (133 bytes)

方式1：声明服务时提供 根模块中提供的服务对象是整个应用中“单例的”
就是上述指令模式创建的服务

方式2：在模块中提供 在当前模块中的所有组件共用同一个服务对象

方式3：在组件声明中提供 该服务仅能作用于当前组件，且每个组件都有自己专有的服务对象
